# Development and Reflection Phase

In this phase, the conceptual design was implemented as the batch-processing data pipeline using Python and a reproducible local Docker environment. The system processes a large-scale e-commerce transaction dataset containing time-referenced records, ensuring that the project aligns with the requirement of handling data-intensive applications. The implementation follows a structured workflow, beginning with data ingestion from the CSV file, where the dataset is loaded and prepared for further processing. This is followed by preprocessing steps such as data cleaning, handling missing values, removing duplicates, type conversion, and feature extraction from timestamp fields, including year, month, and quarter. These steps ensure that the dataset is consistent, accurate, and suitable for further analysis.

After preprocessing, the cleaned data is aggregated to generate meaningful insights. Monthly and quarterly sales summaries are calculated, along with customer segment-based and state-wise insights. These aggregations help in identifying patterns, trends, and business performance indicators over time. Visualizations were produced to monitor trends and evaluate business performance, including graphs for monthly sales, transaction counts, and customer segment distribution. These visual outputs play an important role in understanding the behaviour of the dataset and verifying the correctness of the pipeline.

To extend the functionality of the system, a machine learning model was developed to predict customer spending behaviour using demographic and transactional features. The model was trained using a structured pipeline and evaluated using standard metrics such as Mean Absolute Error, Root Mean Squared Error, and R-squared score. Prediction outputs were generated and compared with actual values to assess model performance.

The implementation aligns with the initial project plan, demonstrating a clear and logical data flow from ingestion to output delivery. Resource considerations were addressed by using Docker for reproducibility and Google Colab for prototype experimentation. The process was continuously monitored through intermediate outputs and visualizations, ensuring correctness and consistency of results. The system successfully demonstrates a working batch-processing pipeline with opportunities for scalability, optimization, and further enhancement.
